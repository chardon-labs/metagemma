Methodology

Confidence-guided trace selection

Gemma 4 E2B was fine tuned to emit a confidence score alongside each generated token by replacing an unused token, without changing the model architecture.

Loss
\[ \begin{aligned} \mathcal{L} &= \lambda_1 D(q_{\mathrm{teacher}}, p_{\theta}) + \lambda_2 H(y, \hat{c}_t) \\ \hat{c}_t &= \sigma(h_t \cdot w_{\mathrm{conf}}) \end{aligned} \]

This maps directly onto inference engines: each sampled trace is just a batch row, while confidence is an auxiliary logit read at every step.

For local models, where inference is often constrained by memory bandwidth, larger batches can amortize weight reads across many candidate traces. This enables policies such as selecting the trace with the highest average confidence.

Sample applications include routing low-confidence generations for review, comparing candidate traces, and visualizing uncertainty across long answers.
